## 1. 브랜드 전략

### 두 개의 축
- **꼬까씬 (kkokkascene)** — 일반 영상 제작 (기존 라인)
- **꼬까에이아이 (KKOKKA.AI STUDIO)** — AI 영상 디렉팅 전문 (이번 프로젝트)
- 도메인: **kkokka.ai** (네임칩, 회사 이메일로 구매)

### 포지셔닝
- AI가 앞이 아니라 **제작 실적이 앞**: "실적 있는 제작사가 여는 AI 라인"
- 신생 AI 업체가 아니라, 검증된 제작팀이 AI를 도구로 쓰는 스튜디오
- 아직 이 자리를 **선점한 브랜드가 없음** → 지금이 기회

### 핵심 관점 (디렉팅론)
- **대체가 아니라 이동**: AI는 손(반복작업)의 시간을 줄이고, 그 시간을 머리(기획·연출·검수)로 옮긴다
- **공공이 증명한다**: 관공서가 AI 영상을 호의적으로 받아들이기 시작 = 검증된 방식이 됐다는 신호
- **안 쓰는 게 손해**: 예산·일정 제약 안에서 불가능하던 장면을 가능하게 함

### 화법 (톤 밸런스)
- 사이트 전면 = **자신감·결과 중심** (절제된 프리미엄 톤)
- "AI가 일자리를 만든다"류의 방어적 담론은 전면에 세우지 않음
- 그 담론은 **인사이트/저널 콘텐츠**에서 깊게 풀기 (미정, 나중 단계)
- 선점자 관점 카피의 강도는 **[미정 — 논의 예정]**

## 7. 관리자 영역 — /admin  ★핵심 요구사항★

> 지난 시도에서 자주 누락됐던 항목들. **반드시 전부 반영.**

### 7-1. 관리자 메모 (좌측바)

- 좌측바 영역에 배치

### 7-2. 즐겨찾기 바 (상단바)
- 관리자 상단에 즐겨찾기(바로가기) 바
- **추가 · 삭제 · 수정** 가능

### 7-3. 좌측 메뉴 = 드롭다운(펼침) 형식
- 대분류 클릭 시 하위 메뉴 펼쳐지는 구조

### 7-4. 회원 조회 ★자주 빠지던 항목★
- **CSV 다운로드**
- **페이징(pagination)**
- **검색/조회 기능**
- → 이 3개는 기본인데 매번 누락됨. **필수 체크.**

### 7-5. 홈 화면 관리
- 홈 **글씨(카피) 수정** 가능
- 홈 **배경 교체** 가능 (특히 영상)
- 영상 업로드가 안 될 경우 **대처법을 화면에 안내** (아래 8번 참조)

### 7-6. 고객 문의 답변 · 비밀글 ★가장 어려운 핵심 기능★
- 1:1 **비밀글** 구조 (작성자와 관리자만 열람)
- 관리자 답변 작성 · 상태 변경 · 내부 메모
- 문의 상태: new / reviewing / quoted / in_progress / closed / spam
- 문의는 **제작문의(Contact)** 화면을 통해 접수 → inquiries 테이블에 비밀글로 저장
- ★**로그인 정책 확정**★: 홈·포트폴리오·프로세스·소개·정책 등은 **전부 비로그인 공개**.
  제작문의 **작성도 로그인 없이 가능**(문턱 낮춤). **답변·견적 확인 시에만 Google 로그인**.
  → 비로그인 작성 시엔 이메일로 문의번호/확인링크를 안내하고, 이후 로그인하면 본인 문의로 연결.
- ★**새 문의가 들어오면 SAYABOUT 메일(sayabout.corp@gmail.com)로 자동 발송**★ (이메일 알림)
  - 방법 후보: Resend / Supabase Edge Function / 웹훅 등 (문의 기능 만들 때 함께 구현)
- ★**공개 정책 확정**★: 목록에는 **제목만** 공개(뷰 inquiries_public). 작성자는 회사명 대신 "AI 제작 문의 드립니다" 같은 일반 제목 사용. **문의 내용·답변은 본인만** 열람. 안내문구: "제작문의 글 및 답변글은 본인만 볼 수 있습니다."
- ※ Supabase RLS(행수준 보안)로 "남의 비밀글이 안 보이게" 반드시 테스트

> 사용자가 개인 채팅방에서 만들어 실사용 중인 `aiee-admin.html`의 레이아웃을 채택.

- **상단바(topbar)**: 로고 + 관리자 이메일/로그아웃 + 바로가기 링크(URL 수정형) + 즐겨찾기 칩(추가·삭제·클릭이동) + 실시간 시계 + 종료 버튼

- **범용 데이터테이블 `dataTable(tid, columns, rows)`**: 검색(전 컬럼) + 컬럼 정렬(▲▼) + 페이지크기(50/100/200) + 페이징(«‹1 2 3›») + **CSV 다운로드(BOM 포함 한글깨짐 방지)**. → 회원조회(7-4)는 이 함수 그대로 사용
- **페이지 head(title, desc)**: 제목 + 설명 + ★즐겨찾기 토글
- **관리자 메모(pgMemo)**: 게시판형 — 목록(번호·제목·작성일) / 새글 / 상세 / 삭제
- **푸터 관리(pgFooter)**: 탭형(하단정보·이용약관·개인정보처리방침·FAQ·B2B문의)
- **toast** 알림, **card / kpi-grid / filter-tabs / pill** 등 공통 컴포넌트 클래스

## 8. 배경 영상 처리

### 비주얼 방향

- 로봇/AI두뇌/과한 네온/사이버펑크 ❌

### fallback (영상이 없거나 로딩 안 될 때)
1. 모바일영상 → 2. PC영상 → 3. 포스터 이미지 → 4. 기본 블랙+애니메이션 그리드 배경
- **영상이 없어도 레이아웃이 깨지지 않아야 함** (현재 HTML 시안에 구현됨)
- `HeroVideo` 컴포넌트가 담당

### 영상 업로드가 안 될 때 대처 (관리자 화면 안내문에 넣을 것)
- 용량이 너무 크면 실패 → 압축본(웹용) 사용 권장
- 대안: 영상 대신 포스터 이미지만 등록해도 정상 동작
- 대안: 외부 URL(예: 스토리지 링크)로 등록하는 방식도 지원

## 12. 작업 방식 & 단계

### 방식
- **통짜로 한 번에 ❌ / 한 줄씩 ❌ → "작동하는 덩어리" 단위 ⭕**
- 각 단계 끝나면 실제로 켜서 확인하고 넘어감
- 앞으로 수정은 "어떤 파일의 어느 부분을 무엇으로" 방식 (통짜 전송 지양)

### 단계 순서 (진행 상황 반영)
1. ✅ **껍데기 배포** — 홈+Works+Process+About+Contact+관리자레이아웃 Vercel 배포 완료
2. ✅ **Supabase 키 연결** (Vercel 환경변수 등록됨)
3. 🔄 **문의 기능** (진행중) — Contact 폼→inquiries 저장 + 관리자에서 목록 조회 + 자동 메일 알림
   - inquiries 테이블 생성됨(3333_inquiries_table.sql), RLS 적용(작성 open / 읽기 본인만)
   - 남은 것: Resend 키 넣어 메일 실제 발송, 배포 후 테스트
4. **Google 로그인** (뒤로 미룸) — 답변·견적 확인용. 문의 작성은 로그인 불필요
5. **문의 답변 주고받기** (7-6 완성) — 로그인 후 본인 문의에 관리자 답변 확인
6. **홈 글씨/영상 관리** (7-5·7-7)
7. **회원/견적/프로젝트/푸터약관** 등 나머지 관리자 기능 (7-4·7-8)
+ 정책 페이지, 마이페이지, 인사이트 등은 이후
